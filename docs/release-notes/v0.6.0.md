# v0.6.0 — Interop exporters + importer coverage + UX fixes

**Uitgebracht:** 2026-04-22

## Hoogtepunten

### 🔁 Exporters — gegevens delen met andere pakketten
Via **Bestand → Exporteren** kun je nu naar drie Nederlandse calculatieformaten exporteren:
- **CUF-XML 4.003** — aanbevolen. Compatibel met Ibis, Kraan, ArchiCalc, WpCalc (~75% van de B&U-markt)
- **IBIS-TRAD XML** — experimenteel; basisstructuur
- **RAW RSX** — CROW GWW; beperkt (1 niveau Deelraming)

**Round-trip tests** garanderen dat import → export → import je data intact laat.

### 📥 Importer-uitbreidingen
- **CUF-XML:** `<Middel>` elementen (arbeid/materiaal/materieel/onderaannemer) worden ingeladen als calculatieregels inclusief `Normfactor` en `Normdeler`. `<Toeslag>`-elementen geven een waarschuwing
- **ZSX prijsbestanden** → ResourceLibrary (nieuwe Import-entry)
- **NSX normenbestanden** → voorlopig geparseerd en gelogd (integratie volgt in v0.7.0)

### 🐛 Bugfixes
- **Nummer-cel** in begrotingsgrid is niet meer klikbaar (was: opent leeg). Waarde blijft zichtbaar
- **PDF Opslaan-als** opent nu in de map van het .ifcCalc-bestand met de begrotingsnaam voorgesteld
- **Staart** percentages volledig bewerkbaar (UKK/AK/W&R + Bouw 1 varianten, 9 regressietests toegevoegd)
- **Spreadsheet**: typen hervat nu direct na Enter-commit — geen klik meer nodig

### 🎨 UX
- **Verticale rasterlijnen** subtieler in alle thema's via nieuwe `--theme-grid-vline`
- **Celopmaak / cel-randen** in spreadsheet: rechtsklik → "Randen…" met 7 presets (Alle, Buitenrand, Dikke buitenrand, Binnenranden, Boven, Onder, Geen)
- **Bouw 1 rapport** toont nu de rapportdatum. Configureerbaar via Projectinformatie; default = vandaag, formaat DD-MM-YYYY

## Technisch

- Nieuwe service-laag `src/services/exporters/` spiegelt `importers/` met gedeeld `xmlBuilder` + Dutch-decimal + unit-denormalisatie
- Canonicalize-helper voor semantische round-trip equivalence
- Chrono dependency in Rust voor datum-formattering
- Typst template in `tenants/bouw1/templates/begroting.typ` rendert `{report_date}`
- `CellBorder` / `CellBorders` types op `SubSheetCell` met snapshot-aware slice actions
- 9 nieuwe vitest tests (exporters + round-trip + staart editability)

## Upgraden
Geen breaking changes; v2.1-bestanden blijven compatibel.

- REST API met OpenAPI
- Normen-integratie in ResourceLibrary (NSX-imports nu alleen geparseerd)
