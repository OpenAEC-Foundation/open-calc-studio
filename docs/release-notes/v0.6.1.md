# v0.6.1 — Live staart-berekening + critical fixes

**Uitgebracht:** 2026-05-01

## Critical fixes

### 🔥 Bouw 1 rapport gebruikt nu live staart-berekening
Eerder werd de staart-uitsplitsing (loon/materiaal/materieel/stelpost/onderaanneming) gecached in `schedule.staartRows` op het moment van WpCalc-import. Wijzigingen aan items na import werden niet doorgerekend → grid en rapport liepen uit elkaar.

**Voor v0.6.1:** AC296 toonde grid 638.448,76 maar rapport 351.204,76 (oude WpCalc-waarde).
**Vanaf v0.6.1:** rapport altijd identiek aan grid. Calculator herrekent de volledige staart-cascade (AK over OA → ABK → garanties → W&P → risico → winst → verzekering → BTW → afronding) op iedere wijziging.

Geïmplementeerd via:
- Nieuwe `StaartItemBreakdown` op elke `staart_*` rij (loon/materiaal/materieel/stelpost/onderaanneming/bedrag/subtotaal/totaal)
- TS calculator `computeStaartItemBreakdowns` met cumulatieve running totals
- `recalculateItems` pijpt automatisch door
- Rust `bouw1.rs` leest items-driven, voegt 4 header-rijen toe ("Totaal kolommen", "Totaal kostprijs", "Totaal excl. btw.", "Totaalprijs incl. btw.")

### 🔥 Infinite re-render loop bij Nieuwe begroting opgelost
Zustand selector in `CostGrid.tsx` retourneerde een nieuwe `[]` op elke render wanneer `schedule.branches` undefined was → React loop. Nu via stabiele module-level constante.
