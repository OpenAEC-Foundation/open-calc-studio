# v0.7.3 — PDF empty-logo decoder fix

**Uitgebracht:** 2026-05-02

## Hotfix

### 🔥 Bouw 1 PDF Export werkte niet zonder logo
Sinds v0.7.1 (brand-neutralisatie) is de hardcoded vendor-logo placeholder verwijderd. De fallback 1x1 PNG had ongeldige CRC's en zlib-stream — Typst kon de bytes niet decoderen, met als gevolg:

```
Typst compile errors: failed to decode image
(Format error decoding Png: Corrupt deflate stream. InvalidUncompressedBlockLength)
```

**Fix:** `EMPTY_PNG` is nu een geverifieerde 68-byte 8-bit gray+alpha 1x1 transparante PNG (bytes gegenereerd via Python zlib).

PDF Export werkt nu out-of-the-box, ook zonder geüploade logo's via Bedrijfsgegevens.

## Compat
Geen wijzigingen aan bestandsformaat. Alle v0.7.x bestanden openen normaal.
